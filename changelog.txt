V2.1 
This is a massive change..
Rewritten the receiver routine to make it more efficient. Receive mode should now be able to handle far more incoming streams for a given set of hardware
Traceroute has been rewritten for OSX/linux - much less CPU intensive
A Raspberry Pi 3 can now happily generate 30Mbps all day without cooking
The 'tx bps' counter on the Transmitter now gives you the 'actual tx bps' (as opposed to the rate you specified) so you know what you're sending. It also puts up a warning if the tx rate is exceeding what you specified at the outset. This gives you some warning that you're probably overloading the hardware.
Packet loss is now to two decimal places
Adds more information to the report - the test duration, the total bytes received, detail about route changes etc.
Also records the TTL value of the incoming packets. This allows a secondary source info about routing changes.
Additional Event types for the report: Traceroute change, Received IP TTL, Source address/port, duplicate packets received, out of order packets received
Help page shows the filenames/paths being saved to
'Burst Mode'  - This will temporarily double the TX rate for a period of 5 secs to allow testing of Mikrotik queues to see what happens when there is a sudden increase in the amount of transmitted data
